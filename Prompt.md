我和我的团队正在为6月8日即将举行的大学 Plug-in Bootcamp 比赛开发一款名为“NeTrix”（Nexus + Matrix）的教育科技MVP。

目前，我们只有大约4周的实际编码时间（不包括期末考试周）。我们有3名技术开发人员（EEE/CS/Math 专业），他们能力很强，但需要一个极其高效、可快速部署的项目模板以便立即开工。

## 产品概念与核心定位
NeTrix 不是传统的校园论坛。它是一个面向高校学生的“AI原生学术共创元宇宙”。我们强调“人-AI-人 (H2A2H)”的工作流，即由AI破冰，人类进行纠正/补充。

### UI/UX设计：赛博神经网络
一个游戏化的“赛博神经网络 (Matrix)”：
- **枢纽 (学院)：** 例如，“商学院”是地图上发光的“主节点”。
- **子节点 (话题)：** 点击一个枢纽将连接到“次级节点”（例如，金融、数学论坛）。
- **视觉风格：** 黑客/赛博朋克风，但保持简洁。默认暗黑模式，辅以霓虹强调色（青色/洋红色）。
- **技术限制：** 使用纯CSS、简单的SVG线条或轻量级库（如 React Flow）来创建这个节点地图UI。**不要**使用重度的3D或图像资源。

### 用户身份：使用协议与装备代替虚拟形象
我们**不使用**传统的RPG“虚拟形象”（这会破坏赛博朋克的沉浸感）。取而代之的是，我们使用一套“协议 (Protocols)”和“插件 (Plugins)”的游戏化系统：
1. **切换协议 (状态)：** 用户切换其网络协议以显示意图。
   - 例如，协议：探索者（蓝色节点，寻找答案）。
   - 例如，协议：先知（发光的金色节点，准备回答问题）。
2. **装备插件 (游戏化)：** 用户根据自己的专长装备“数字插件”。
   - 例如，[高级微积分引擎 v2.0]，[宏观经济数据挖掘器]。

### 要构建的3个核心模块
1. **知识中心 (节点论坛)：** - 我们想要一个以人为本的、真正的 StackOverflow 风格的环境。AI**不应**喧宾夺主。
   - **“召唤”协议 (选择性AI)：** AI Agent **不会**自动回复每个帖子。只有当用户明确点击“召唤先知”按钮或输入 @Oracle 时，它才会回答。
   - **真正的对等智能体 (竞技场)：** 当AI回答时，它的帖子表现得和普通人类的回复完全一样。它与人类的答案并列展示。
   - AI和人类的答案都受到相同的**赞成/反对 (Upvote/Downvote)** 系统的约束。社区决定哪个答案能浮到顶部，确保碳基和硅基用户之间真正的平等。
2. **学习搭子 (匹配 -> 节点共振)：** - **重要提示：** 我们**已经有**一个在之前 UNNC 30H AI Hackathon 比赛中写好的有效匹配算法。我们**不需要**伪造这个功能。
   - **集成方式：** 当用户点击“启动雷达扫描”时，前端应显示一个CSS雷达脉冲动画（加载状态）。同时，它会向封装了我们自定义算法的后端/边缘函数 (Edge Function) 发起异步调用。
   - **数据流：** API接收用户的 [插件] 和 [协议] 数组，在数据库中运行算法，并返回匹配度最高的3个用户ID。然后UI会在地图上高亮显示这些特定的节点。
- 既然我们已经有了现成的算法（可能是Python或Node.js），请建议将其与BaaS集成的最佳架构（例如，Supabase Edge Functions 与 Render 上独立的 FastAPI 微服务对比）。
- 请提供此“雷达扫描”匹配端点的精确API路由结构（请求/响应的Payload设计）。
3. **资源中心 (知识库)：** - 结构化的历年试卷、学习资料以及其他全校性新闻。
   - **“AI低语”画廊：** 一个特定版块，用户可以在此分享他们与AI高质量、深度对话的链接（Prompt分享）。

## 严格的开发限制
- 时间紧迫。我们**不能**从零开始构建后端（不使用 SpringBoot，不使用 Django）。
- 我们必须使用BaaS（后端即服务），如 Supabase、Firebase 或腾讯云开发来实现身份验证、数据库和边缘函数。
- UI应严重依赖预构建的组件库（React/Vue + Tailwind, shadcn/ui 等）以节省时间。对于MVP，2.5D地图可以只是一个带有可点击热区的静态图片。
